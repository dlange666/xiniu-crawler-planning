**写作约定**

- 文件名：kebab-case，**不带版本号**（`infra-fetch-policy.md` 而不是
  `infra-fetch-policy-v1.md`）。版本号统一靠 `## 修订历史` 与 git 历史。
- 顶部 frontmatter 用引用块（`> ...`）；不用 YAML，避免渲染歧义。
- 章节编号用 `##` 起 §1，依次递增；不在 spec 内部嵌套 4 级以上标题。
- 数据表格优先于散文式描述；散文用于阐述决策与权衡。
- 结尾必须有 "不在本 spec 范围"，避免无限解释边界问题。
- 若引用其它 spec，写相对路径：``docs/prod-spec/<name>.md``。
