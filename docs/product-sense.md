# 产品方向

## 目标

围绕**烯牛政策图谱**：聚合海量产业政策，通过 AI 深度结构化处理，
形成"检索 + 解读 + 统计 + 对比 + 推送"的政策数据服务。

爬虫仓库承担其中**"政策采集 + 原始留存 + 结构化标签 → 标准 JSON"** 链路。

## 用户与价值

- 终端用户：LP、GP、银行、券商、企业战投/研究
- 我们的交付物：稳定、可回放、可审计、可订阅的政策数据集

## 核心指标（爬虫侧）

| 维度 | 指标 | MVP 门槛 |
|---|---|---|
| 覆盖 | 第一阶段数据源已配置数 | 105+ 种入口 |
| 鲜度 | 新政策从发布到入库 P95 时延 | < 24h |
| 保真 | 原始页可回放比例 | = 100% |
| 解析 | AI 结构化 JSON schema 合格率 | ≥ 90% |
| 礼貌 | host 维度 429/5xx 比例 | < 1% |
| 合规 | robots / 反爬命中后未尝试绕过 | = 100% |

## 不做什么（边界）

- 不做绕过验证码/登录/付费墙/技术 challenge 的能力
- 不做新闻聚合站的全网络爬取（仅限官方机关与协会）
- 第一阶段不做用户侧前端（前端形态在 `docs/prd/policy-graph-product-plan-20260427.md` 中由产品同学规划）
- 第一阶段不做基于 simhash 的相似政策自动合并
